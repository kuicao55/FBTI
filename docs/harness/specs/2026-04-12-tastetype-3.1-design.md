# TasteType 3.1 升级设计文档

**Date:** 2026-04-12
**Status:** Draft

## Goal

将 FBTI 从当前版本升级至 TasteType 3.1：更新人格模型（80类型）、题库（100题）、评分算法（分层抽样+累积百分比）、结果页（诙谐自述+味觉签名+详细解读+雷达图）。

## Architecture

单页 Vanilla JS 应用（无框架），保持现有 CSS 设计系统，仅替换数据层和逻辑层。

## 数据结构变更

### 1. 题目数据 `data/questions.json`

- **结构变更**：从 16 题固定顺序 → 100 题分层抽样池
- **单次测试**：24 题（第一维度6题 + 第二维度8题 + 第三维度4题 + 第四维度6题）
- **每题字段**：`id`, `text`, `dimension`, `options[]`（`label`, `text`, `scores`对象）

```js
// 示例
{
  "id": 1,
  "text": "朋友点了一盘红油满满的菜，你的第一反应是？",
  "dimension": "stimulus", // stimulus | taste | philosophy | novelty
  "options": [
    { "label": "A", "text": "直接夹一筷子，先尝尝辣得正不正", "scores": { "H": 1 } },
    { "label": "B", "text": "先观察一下有没有花椒，有花椒更兴奋", "scores": { "N": 1 } },

    { "label": "D", "text": "内心默默祈祷不要太辣", "scores": { "M": 1 } }
  ]

```

### 2. 人格类型数据 `data/types.json`

- **结构变更**：从 16 类型 → 80 类型
- **每类型字段**：`name`, `selfDescription`, `tasteSignature`, `analysis`, `dimensions`

```js

```

## 评分算法

### 分层随机抽样

每次测试从100题库中按维度分层抽取24题：

| 维度 | 代码 | 题库量 | 抽取量 | 保底规则 |
|------|------|--------|--------|----------|
| 第一维度 | H/N/C/M | 24 | 6 | 每种至少1题 |
| 第二维度 | U/S/W/B/O | 30 | 8 | 每种至少1题 |
| 第三维度 | A/S | 20 | 4 | 每种至少1题 |
| 第四维度 | E/C | 26 | 6 | 每种至少2题 |

**实现逻辑**：
1. 对每个维度，按保底规则优先抽取（如第一维度：H/N/C/M各抽1题，剩余2题随机）
2. 剩余题目从该维度题库随机补充
3. 合并四维度题目后打乱顺序

### 累积计分制

每题选项携带多个维度的分数（对象形式），累加各维度得分后计算百分比。

```js
// 示例选项
{ "label": "A", "text": "...", "scores": { "H": 1 } }
{ "label": "B", "text": "...", "scores": { "M": 0.5, "N": 0.5 } }
```

### 百分比与主导字母

每个维度独立计算：
- **该维度总得分** = 所有选项得分之和
- **各代码百分比** = (该代码得分 ÷ 维度总得分) × 100%
- **主导字母**：百分比最高的代码
- **次级字母**：若第二高 ≥ 主导×70%，则附加小写字母

### 平局处理

若出现同分最高项，从该维度题库抽取一道「二选一迫选题」打破平局，次级字母自动记录另一项。

## 题目抽取实现

```js
function selectQuestions(questionsData) {
  const { stimulus, taste, philosophy, novelty } = groupByDimension(questionsData);

  const selected = [
    ...ensureMinPerTendency(stimulus, 6, ['H','N','C','M'], [1,1,1,1]),
    ...ensureMinPerTendency(taste, 8, ['U','S','W','B','O'], [1,1,1,1,1]),
    ...ensureMinPerTendency(philosophy, 4, ['A','S'], [1,1]),
    ...ensureMinPerTendency(novelty, 6, ['E','C'], [2,2]),
  ];

  return shuffle(selected);
}
```

## 结果页 UI

### 主显示（默认可见）

- **头像**：圆形占位符，后续替换为真实图片
- **类型代码（主）**：`HUAE`（纯大写，无连字符）
- **类型代码（次）**：`Hn-Ub-A-E`（小写+连字符，显示在主代码下方，opacity: 0.4）
- **人格昵称**：如"山野炼金士"
- **味觉签名标签**：四枚标签横向排列，如 `H 灼烧` `U 鲜味` `A 加法烹饪` `E 探索者`
- **诙谐自述**：带左侧橙色竖线的内容块
- **详细人格解读**：2-3段文字描述

### 详情面板（折叠展开）

点击「查看详细分析」展开：

- **雷达图**：CSS/SVG 绘制，四轴（刺激偏好、味觉引力、风味哲学、探索指数），每轴显示主导字母百分比
- **分段进度条**：每个维度一个条，每种字母一个颜色分段（从大到小排列）
  - 图例：圆点 + 字母 + 字母含义 + 百分比
  - 例：`● H(灼烧) 72%`

### 分享图片

Canvas 生成的 PNG 保持现有布局，但更新内容字段（`name`→人格昵称，`tagline`→诙谐自述）。

## 雷达图实现

纯 SVG，无外部依赖：

- 同心圆背景（3层，opacity递减）
- 三条轴线（中心到边缘）
- 数据多边形（fill+stroke）
- 每轴端点圆点
- 轴标签文字

## 边界情况

| 情况 | 处理方式 |
|------|----------|
| 分层抽样某倾向题库不足 | 用该维度其他相近倾向题目替代 |
| 某维度平局 | 从该维度题库随机抽一道「二选一」打破平局 |
| 所有题目全选同一倾向 | 判定为无效作答，提示重新测试 |
| 用户中途退出 | 已答题目保留，重新进入从断点继续 |
| 短时间重复测试 | 题目随机抽取，保证至少40%不同 |
