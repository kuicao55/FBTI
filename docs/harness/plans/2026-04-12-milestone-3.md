# Milestone-3: Update Data Layer (questions + types) Implementation Plan

> **Harness note:** This plan is executed via `harness-execution` using the Orchestrator / Executor / Reviewer architecture. Each task goes through Executor (TDD implementation) → Spec Reviewer (compliance check) → Code Quality Reviewer (adversarial review). Only Code Quality Review PASS closes a task.

**Goal:** Replace questions.json (100 questions) and types.json (80 personality types) with TasteType 3.1 data.

**Milestone ref:** milestone-3 from claude-progress.json

## Task 1: Create data/questions.json

**Files:**
- Create: `data/questions.json`

**TDD_EVIDENCE:** `cat data/questions.json | python3 -c "import json,sys; d=json.load(sys.stdin); print(f'Questions: {len(d[\"questions\"])}')"` should output "Questions: 100".

- [ ] **Step 1: Write data/questions.json**

```json
{
  "dimensions": ["stimulus", "taste", "philosophy", "novelty"],
  "questions": [
    {
      "id": 1,
      "text": "朋友点了一盘红油满满的菜，你的第一反应是？",
      "dimension": "stimulus",
      "options": [
        { "label": "A", "text": "直接夹一筷子，先尝尝辣得正不正", "scores": { "H": 1 } },
        { "label": "B", "text": "先观察一下有没有花椒，有花椒更兴奋", "scores": { "N": 1 } },

        { "label": "D", "text": "内心默默祈祷不要太辣", "scores": { "M": 1 } }
      ]
    }
    // ... Q2-Q100 from FBTI_3.1_Questions.md
  ]

```

- [ ] **Step 2: Validate data/questions.json**

Expected: "Questions: 100, Dims: ['stimulus', 'taste', 'philosophy', 'novelty']"

- [ ] **Step 3: Verify dimension distribution**

- [ ] **Step 4: Commit**
