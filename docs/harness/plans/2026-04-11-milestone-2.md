# Modularize FBTI into Independent Modules Implementation Plan

> **Harness note:** This plan is executed via `harness-execution` using the Orchestrator / Executor / Reviewer architecture. Each task goes through Executor (TDD implementation) → Spec Reviewer (compliance check) → Code Quality Reviewer (adversarial review). Only Code Quality Review PASS closes a task.

**Goal:** 将 FBTI 单文件应用拆分为独立模块 (data/, modules/), 支持人格分类、题库、评分逻辑独立更新。

**Milestone ref:** milestone-2 from claude-progress.json

**Architecture:** 5层架构 (数据层 → 接口层 → 引擎层 → 渲染层 → 入口层)。使用原生 ES Module + JSON,无需构建工具。浏览器直接支持 ES Module import/export。

**Tech Stack:** Vanilla JavaScript (ES Module), JSON, SVG

---

## File Structure

| File                    | Action | Responsibility                              |
| ----------------------- | ------ | ------------------------------------------- |
| `data/types.json`       | Create | 16种人格分类定义,包含 dimensions 字段       |
| `data/questions.json`   | Create | 16道题目,包含 dimension 和选项分值           |
| `modules/loader.js`     | Create | 统一加载 data/*.json,提供缓存                |
| `modules/scoring.js`    | Create | 评分引擎,接收 answers/questions/types       |
| `modules/render.js`     | Create | 渲染引擎,题目渲染+结果渲染+SVG加载           |
| `index.html`            | Modify | 移除内联数据/逻辑,改为 import 模块,保留 UI   |
| `assets/*.svg`          | Keep   | 保持现有16个SVG头像文件,无需修改              |
